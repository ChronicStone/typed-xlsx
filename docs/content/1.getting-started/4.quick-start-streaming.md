title: Quick Start — Streaming
description: Export hundreds of thousands of rows with flat memory usage using createWorkbookStream().
---

Streaming mode writes row batches to a spool on disk as they arrive — no rows are held in memory after each commit. Use it when your dataset is too large to hold in memory, or when rows come from a database cursor or paginated API.

## When to use streaming

- Exports with > 100k rows
- Rows come from a database cursor, paginated endpoint, or async generator
- You need to keep Node.js heap usage flat regardless of dataset size

For smaller datasets, [`createWorkbook()`](/getting-started/quick-start-buffered) is simpler.

## Define the schema

Schemas work identically in buffered and streaming mode — build it once, use it in either:

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";

type OrderLine = {
  orderId: string;
  sku: string;
  productName: string;
  qty: number;
  unitPrice: number;
  warehouseRegion: string;
  shippedAt: string | null;
};

const orderSchema = createExcelSchema<OrderLine>()
  .column("orderId", { header: "Order ID", accessor: "orderId", width: 14 })
  .column("sku", { header: "SKU", accessor: "sku", width: 12 })
  .column("productName", {
    header: "Product",
    accessor: "productName",
    minWidth: 24,
    autoWidth: true,
  })
  .column("qty", { header: "Qty", accessor: "qty", width: 8 })
  .column("unitPrice", {
    header: "Unit Price",
    accessor: "unitPrice",
    width: 14,
    style: { numFmt: "$#,##0.00" },
  })
  .column("lineTotal", {
    header: "Line Total",
    formula: ({ row, fx }) => fx.round(row.ref("qty").mul(row.ref("unitPrice")), 2),
    width: 14,
    style: { numFmt: "$#,##0.00" },
    summary: (s) => [s.formula("sum")],
  })
  .column("warehouseRegion", { header: "Region", accessor: "warehouseRegion", width: 10 })
  .column("shippedAt", {
    header: "Shipped",
    accessor: "shippedAt",
    defaultValue: "—",
    width: 14,
  })
  .build();
```

## Stream rows in batches

```ts twoslash
import { createExcelSchema, createWorkbookStream } from "@chronicstone/typed-xlsx";

type OrderLine = {
  orderId: string;
  sku: string;
  productName: string;
  qty: number;
  unitPrice: number;
  warehouseRegion: string;
  shippedAt: string | null;
};

const orderSchema = createExcelSchema<OrderLine>()
  .column("orderId", { accessor: "orderId" })
  .column("sku", { accessor: "sku" })
  .column("productName", { accessor: "productName" })
  .column("qty", { accessor: "qty" })
  .column("unitPrice", { accessor: "unitPrice" })
  .column("lineTotal", {
    formula: ({ row, fx }) => fx.round(row.ref("qty").mul(row.ref("unitPrice")), 2),
  })
  .column("warehouseRegion", { accessor: "warehouseRegion" })
  .column("shippedAt", { accessor: "shippedAt" })
  .build();
// ---cut---
const workbook = createWorkbookStream();

// Create the table — no rows yet
const table = await workbook.sheet("Orders", { freezePane: { rows: 1 } }).table("orders", {
  schema: orderSchema,
});

// Simulate paginated DB fetch — in practice, replace with your cursor/query
async function* fetchOrderPages(): AsyncGenerator<OrderLine[]> {
  yield [
    {
      orderId: "ORD-0001",
      sku: "WIDGET-A",
      productName: "Widget A",
      qty: 50,
      unitPrice: 14.99,
      warehouseRegion: "EMEA",
      shippedAt: "2025-02-01",
    },
    {
      orderId: "ORD-0002",
      sku: "GADGET-B",
      productName: "Gadget B",
      qty: 12,
      unitPrice: 89.99,
      warehouseRegion: "AMER",
      shippedAt: null,
    },
  ];
  // yield next page...
}

for await (const page of fetchOrderPages()) {
  await table.commit({ rows: page });
  // Memory freed after each commit — heap doesn't grow with total row count
}

// Assemble and write — streams spool → ZIP → file
await workbook.writeToFile("./orders-export.xlsx");
```

## Using an excel-table schema

Excel-table schemas work in streaming mode with no API changes:

```ts twoslash
import { createExcelSchema, createWorkbookStream } from "@chronicstone/typed-xlsx";

type Sale = { region: string; amount: number; units: number };

const saleSchema = createExcelSchema<Sale>({ mode: "excel-table" })
  .column("region", { header: "Region", accessor: "region" })
  .column("amount", {
    header: "Amount",
    accessor: "amount",
    style: { numFmt: "$#,##0.00" },
    totalsRow: { function: "sum" },
  })
  .column("units", {
    header: "Units",
    accessor: "units",
    totalsRow: { function: "sum" },
  })
  .build();

const workbook = createWorkbookStream();
const table = await workbook.sheet("Sales").table("sales", {
  schema: saleSchema,
  totalsRow: true,
  style: "TableStyleMedium6",
});

await table.commit({ rows: [{ region: "AMER", amount: 62000, units: 415 }] });
await table.commit({ rows: [{ region: "EMEA", amount: 45000, units: 301 }] });

await workbook.writeToFile("./sales.xlsx");
```

## Memory tuning

Pass options to `createWorkbookStream()` to control the spool and string encoding:

```ts twoslash
import { createWorkbookStream } from "@chronicstone/typed-xlsx";

const workbook = createWorkbookStream({
  tempStorage: "file", // "file" (default) or "memory"
  strings: "inline", // "shared" (default, compact file) or "inline" (faster, larger file)
  memoryProfile: "low-memory", // preset: "balanced" | "low-memory" | "compact-file"
});
```

See [Memory Tuning](/streaming/memory-tuning) for the full reference.
