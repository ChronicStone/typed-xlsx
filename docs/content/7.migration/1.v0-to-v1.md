# Migrating from v0.x to v1.0

v1 is a full rewrite. The XLSX serialization engine, the ZIP assembler, and the streaming pipeline are all new. The public API has been redesigned to match — cleaner names, a consistent builder style, and the new `createWorkbookStream()` entry point. This page covers every breaking change with before/after examples.

## Why the rewrite?

v0 was built on `xlsx-js-style`, which serializes the entire workbook into memory before producing output. That boundary made streaming impossible and created a dependency on a library that was itself a fork of SheetJS. v1 ships its own OOXML serialization and ZIP assembly layer, which enables the stream builder to process rows one batch at a time and produce output incrementally.

## Entry points

### Schema builder

```ts
// v0
import { ExcelSchemaBuilder } from "@chronicstone/typed-xlsx";
const schema = ExcelSchemaBuilder.create<Row>()
  .column(...)
  .build();

// v1
import { createSchema } from "@chronicstone/typed-xlsx";
const schema = createSchema<Row>()
  .column(...)
  .build();
```

### Workbook builder

```ts
// v0
import { ExcelBuilder } from "@chronicstone/typed-xlsx";
const workbook = ExcelBuilder.create();

// v1
import { createWorkbook } from "@chronicstone/typed-xlsx";
const workbook = createWorkbook();
```

## Column definition changes

### `key` → `accessor`

The property that points to the row data was renamed from `key` to `accessor`. It now also accepts a callback for computed values, removing the need for `transform` in most cases.

```ts
// v0 — string path only
.column("email", { key: "user.email" })

// v1 — string path or callback
.column("email", { accessor: "user.email" })
.column("fullName", { accessor: (row) => `${row.firstName} ${row.lastName}` })
```

### `label` → `header`

```ts
// v0
.column("name", { key: "name", label: "Full Name" })

// v1
.column("name", { accessor: "name", header: "Full Name" })
```

### `default` → `defaultValue`

```ts
// v0
.column("score", { key: "score", default: 0 })

// v1
.column("score", { accessor: "score", defaultValue: 0 })
```

### `cellStyle` → `style`

```ts
// v0
.column("amount", {
  key: "amount",
  cellStyle: { numFmt: "$#,##0.00" },
})

// v1
.column("amount", {
  accessor: "amount",
  style: { numFmt: "$#,##0.00" },
})
```

`headerStyle` is unchanged.

### `CellStyle` shape changes

v0 used `xlsx-js-style`'s style shape, which required nested font objects with specific keys. v1 uses its own `CellStyle` type with a cleaner shape.

```ts
// v0 (xlsx-js-style shape)
cellStyle: {
  font: { bold: true, color: { rgb: "FF0000" } },
  fill: { fgColor: { rgb: "FFFF00" } },
  alignment: { horizontal: "center" },
}

// v1 (CellStyle shape)
style: {
  font: { bold: true, color: { rgb: "FF0000" } },
  fill: { color: { rgb: "FFFF00" } },   // fgColor → color, directly on fill
  alignment: { horizontal: "center" },
}
```

Key fill change: `fill.fgColor.rgb` is now `fill.color.rgb`.

## `withTransformers()` and `withFormatters()` removed

In v0, shared transform and format functions were registered on the schema with `.withTransformers()` and `.withFormatters()`, then referenced by name in each column.

```ts
// v0
const schema = ExcelSchemaBuilder.create<Row>()
  .withTransformers({
    uppercase: (value: string) => value.toUpperCase(),
  })
  .column("name", { key: "name", transform: "uppercase" })
  .build();
```

In v1, pass the function directly in the column definition. Shared helpers are just plain TypeScript functions.

```ts
// v1
const uppercase = (value: string) => value.toUpperCase();

const schema = createSchema<Row>()
  .column("name", { accessor: "name", transform: (value) => uppercase(value) })
  .build();
```

## Summary API

v0 summaries received the full `T[]` array via a `value` function and returned a cell value directly.

```ts
// v0
.column("amount", {
  key: "amount",
  summary: [
    {
      value: (data) => data.reduce((sum, row) => sum + row.amount, 0),
      format: "$#,##0.00",
      cellStyle: { font: { bold: true } },
    },
  ],
})
```

v1 uses a reducer model: `init` / `step` / `finalize`. This is what makes stream mode possible — the accumulator is updated per batch without holding all rows.

```ts
// v1
.column("amount", {
  accessor: "amount",
  summary: {
    init: () => 0,
    step: (acc, row) => acc + row.amount,
    finalize: (acc) => acc,
    format: "$#,##0.00",
    style: { font: { bold: true } },
  },
})
```

Multiple summaries: pass an array of `SummaryDefinition` objects — the same as v0's array format.

Note also: `cellStyle` on the summary definition is now `style`.

## Column selection

v0 used a column map where each key was set to `true` or `false`.

```ts
// v0
.addTable({
  schema,
  data: rows,
  select: { name: true, email: true, phone: false },
})
```

v1 uses explicit `include` or `exclude` arrays, keyed by the column `id` (what was `columnKey` in v0).

```ts
// v1
.table({
  schema,
  rows,
  select: { include: ["name", "email"] },
  // or: select: { exclude: ["phone"] },
})
```

## Table input changes

### `.addTable()` → `.table()`

The method on the sheet builder was renamed.

```ts
// v0
workbook.sheet("Report").addTable({ schema, data: rows });

// v1
workbook.sheet("Report").table({ schema, rows });
```

### `data` → `rows`

The row array was renamed from `data` to `rows`.

---

## Build / output changes

v0 `.build()` took a required `output` parameter that determined the return type.

```ts
// v0
const buffer = ExcelBuilder.create()
  .sheet("Data")
  .addTable({ schema, data: rows })
  .build({ output: "buffer" });

const uint8 = ...build({ output: "uint8array" });
await ...build({ output: "file", filePath: "./report.xlsx" });
```

v1 separates output into explicit methods called after the workbook is fully built. There is no longer a required parameter to `.build()`.

```ts
// v1
const workbook = createWorkbook().sheet("Data").table({ schema, rows });

const buffer = workbook.toBuffer();
const uint8 = workbook.toUint8Array();
await workbook.writeToFile("./report.xlsx");
```

## Sheet options

v0 sheet options like `tablesPerRow`, `rowHeight`, and `extraLength` (column gap) have been renamed and reorganized.

| v0                         | v1                                              |
| -------------------------- | ----------------------------------------------- |
| `tablesPerRow`             | `tablesPerRow` (unchanged)                      |
| `extraLength` (column gap) | `tableColumnGap`                                |
| `rowHeight`                | removed — per-row height is not supported in v1 |
| `rtl` on `.build()`        | `rightToLeft` on `.sheet()`                     |

```ts
// v0
ExcelBuilder.create()
  .sheet("Report", { tablesPerRow: 2 })
  .addTable(...)
  .build({ output: "buffer", rtl: true });

// v1
createWorkbook()
  .sheet("Report", { tablesPerRow: 2, tableColumnGap: 2, rightToLeft: true })
  .table(...)
  .toBuffer();
```

## Dynamic column groups

The `.group()` API was carried forward with the same intent, but the registration mechanism changed. In v0, context was inferred from what was registered on the schema. In v1, the context type is explicit at group definition time.

```ts
// v0
schema.group("dates", (builder, ctx: Record<string, string>) => {
  for (const [key, label] of Object.entries(ctx)) {
    builder.column(key, { key: () => ..., label })
  }
})

// v1
createSchema<Row>()
  .group<Record<string, string>>("dates", (builder, ctx) => {
    for (const [key, label] of Object.entries(ctx)) {
      builder.column(key, { accessor: () => ..., header: label })
    }
  })
  .build()
```

Context is passed at table creation time via the `context` field, keyed by group id — this is the same as v0.

## Stream workbook

`createWorkbookStream()` is new in v1. There is no equivalent in v0. See the [Stream Workbook](/stream-workbook/overview) section for full documentation.

## Removed exports

| v0 export            | Status in v1                             |
| -------------------- | ---------------------------------------- |
| `ExcelSchemaBuilder` | Removed — use `createSchema<T>()`        |
| `ExcelBuilder`       | Removed — use `createWorkbook()`         |
| `TransformersMap`    | Removed — no longer needed               |
| `FormattersMap`      | Removed — no longer needed               |
| `FormatterPreset`    | Removed — no longer needed               |
| `TOutputType`        | Removed — output is via explicit methods |
| `ExcelBuildParams`   | Removed                                  |
| `ExcelBuildOutput`   | Removed                                  |
| `SheetTableBuilder`  | Removed                                  |

The `CellStyle` type is still exported. Its shape changed (see above).

## Quick reference

| Change            | v0                                     | v1                                    |
| ----------------- | -------------------------------------- | ------------------------------------- |
| Schema factory    | `ExcelSchemaBuilder.create<T>()`       | `createSchema<T>()`                   |
| Workbook factory  | `ExcelBuilder.create()`                | `createWorkbook()`                    |
| Column field path | `key`                                  | `accessor`                            |
| Column header     | `label`                                | `header`                              |
| Column fallback   | `default`                              | `defaultValue`                        |
| Column style      | `cellStyle`                            | `style`                               |
| Fill color        | `fill.fgColor.rgb`                     | `fill.color.rgb`                      |
| Row array field   | `data`                                 | `rows`                                |
| Add table         | `.addTable()`                          | `.table()`                            |
| Select columns    | `{ col: true/false }`                  | `{ include: [] }` / `{ exclude: [] }` |
| Summary reducer   | `value: (data) => ...`                 | `init / step / finalize`              |
| Summary style     | `cellStyle`                            | `style`                               |
| RTL option        | `.build({ rtl })`                      | `.sheet(name, { rightToLeft })`       |
| Column gap        | `extraLength`                          | `tableColumnGap`                      |
| Buffer output     | `.build({ output: "buffer" })`         | `.toBuffer()`                         |
| Uint8Array output | `.build({ output: "uint8array" })`     | `.toUint8Array()`                     |
| File output       | `.build({ output: "file", filePath })` | `.writeToFile(path)`                  |
