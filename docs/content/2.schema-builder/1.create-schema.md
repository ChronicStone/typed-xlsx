---
title: Create Schema
description: How createSchema() works and what it returns.
---

Schemas are created with `createExcelSchema()`. A schema defines the columns, summaries, styling, and accessors that a table will use.

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";

type User = {
  firstName: string;
  lastName: string;
};

const schema = createExcelSchema<User>();
```

Like the previous API, the builder is chainable and duplicate column ids are rejected early:

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";

const schema = createExcelSchema<{ firstName: string; lastName: string }>()
  .column("firstName", { accessor: "firstName" })
  .column("lastName", { accessor: "lastName" })
  .build();
```

The major naming change is that columns now have:

- an `id`, which is the first argument to `.column(...)`
- an `accessor`, which can be a typed path or a callback

That split keeps column identity stable while letting extraction stay flexible.
