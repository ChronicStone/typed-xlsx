---
title: Create Schema
description: How createExcelSchema() works and what it returns.
---

Schemas are created with `createExcelSchema()`. A schema defines the columns, summaries, styling, and accessors that a table will use.

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";

type User = {
  firstName: string;
  lastName: string;
};

const schema = createExcelSchema<User>();
```

Like the previous API, the builder is chainable and duplicate column ids are rejected early:

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";

const schema = createExcelSchema<{ firstName: string; lastName: string }>()
  .column("firstName", { accessor: "firstName" })
  .column("lastName", { accessor: "lastName" })
  .build();
```

The major naming change is that columns now have:

- an `id`, which is the first argument to `.column(...)`
- an `accessor`, which can be a typed path or a callback

That split keeps column identity stable while letting extraction stay flexible.

## Column IDs accumulate in the type

Every `.column()` call threads the column's `id` as a string literal into the builder's type. By the time you call `.build()`, the schema carries the complete union of all declared IDs. Hover over `ColumnIds` to see it:

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";
import type { SchemaColumnId } from "@chronicstone/typed-xlsx";

const schema = createExcelSchema<{ name: string; email: string; role: string }>()
  .column("name", { accessor: "name" })
  .column("email", { accessor: "email" })
  .column("role", { accessor: "role" })
  .build();

type ColumnIds = SchemaColumnId<typeof schema>;
//   ^?
```

This union is what powers the compile-time validation of `select.include` and `select.exclude` — there are no magic string comparisons at runtime.

## Duplicate IDs are rejected at runtime

Passing the same id twice throws immediately on schema construction, before any row data is processed:

```ts twoslash
// @errors: 2578
import { createExcelSchema } from "@chronicstone/typed-xlsx";

// Throws: Column with id 'name' already exists.
createExcelSchema<{ name: string }>()
  .column("name", { accessor: "name" })
  // @ts-expect-error - duplicate id
  .column("name", { accessor: "name" })
  .build();
```
