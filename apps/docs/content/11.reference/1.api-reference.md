## `createExcelSchema<T>(options?)`

Returns a schema builder. Schemas carry no mutable state after `.build()` and can be reused across any number of tables.

- `createExcelSchema<T>()` returns the default report-schema builder
- `createExcelSchema<T, TContext>()` declares a schema-level context object available to `dynamic(...)`, accessors, formulas, styles, transforms, formats, and hyperlinks
- `createExcelSchema<T>({ mode: "report" })` also returns the report-schema builder
- `createExcelSchema<T>({ mode: "excel-table" })` returns the native Excel table schema builder
- both schema builders also support `.theme(theme)` to attach a reusable base spreadsheet theme to the built schema

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";
type Row = { name: string };
// ---cut---
const schema = createExcelSchema<Row>().column("name", { accessor: "name" }).build();
```

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";
type Row = { amount: number; name: string };
// ---cut---
const schema = createExcelSchema<Row>({ mode: "excel-table" })
  .column("name", { accessor: "name" })
  .column("amount", { accessor: "amount" })
  .build();
```

### `SchemaBuilder.column(id, definition)`

Report-schema builder.

| Field              | Type                                                                | Description                                                                          |
| ------------------ | ------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| `id`               | `string`                                                            | Stable column identifier. Must be unique within the schema.                          |
| `accessor`         | `Path<T>` or `(row: T) => value`                                    | Extracts the cell value from the row.                                                |
| `header`           | `string \| (() => string)`                                          | Column header text. Defaults to `id`.                                                |
| `condition`        | `({ ctx }) => boolean`                                              | Schema-structure predicate. When it returns `false`, the column is omitted entirely. |
| `defaultValue`     | `CellValue`                                                         | Fallback when accessor returns `null` or `undefined`.                                |
| `transform`        | `({ value, row, rowIndex, ctx }) => CellValue`                      | Post-processes the extracted value.                                                  |
| `format`           | `string` or `({ row, rowIndex, subRowIndex, ctx }) => string`       | Excel number format string applied to the cell.                                      |
| `style`            | `CellStyle` or `({ row, rowIndex, subRowIndex, ctx }) => CellStyle` | Style for data cells.                                                                |
| `hyperlink`        | `string` or `{ target, tooltip?, style? }` or callback              | Native Excel hyperlink metadata for the cell without changing its value.             |
| `conditionalStyle` | `(builder) => builder.when(...)` or rule array                      | Native Excel conditional formatting rules for the column range.                      |
| `validation`       | `(builder) => builder.integer().between(...)` or rule object        | Native Excel data validation for the column range.                                   |
| `headerStyle`      | `CellStyle`                                                         | Style for the header cell.                                                           |
| `width`            | `number`                                                            | Column width in character units.                                                     |
| `autoWidth`        | `boolean`                                                           | Compute width from content. Overridden by explicit `width`.                          |
| `minWidth`         | `number`                                                            | Minimum width when `autoWidth` is active.                                            |
| `maxWidth`         | `number`                                                            | Maximum width when `autoWidth` is active.                                            |
| `expansion`        | `"auto" \| "single" \| "expand"`                                    | Controls how formula columns render across expanded physical rows.                   |
| `summary`          | `SummaryInput<T>`                                                   | Reducer-based summary cells, defined directly or via the callback builder.           |

### `SchemaBuilder.group(id, buildFn)`

Report-schema builder. The same flat column-group API is also available on `ExcelTableSchemaBuilder`.

Creates a structural scope with stable child columns.

You can also pass `SchemaBuilder.group(id, options, buildFn)`.

| Argument  | Type                                                      | Description                                                                              |
| --------- | --------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| `id`      | `string`                                                  | Structural group identifier. Used in `select` and formula scope selectors.               |
| `options` | `{ header?: LazyText; condition?: ({ ctx }) => boolean }` | Optional structural metadata. `condition` omits the whole group when it returns `false`. |
| `buildFn` | `(builder: SchemaBuilder<T>) => void`                     | Called with a child builder for defining the group's child columns.                      |

`condition` is structural: it decides whether the node exists in the built table shape at all. It is evaluated from schema context only, not from row data.

### `SchemaBuilder.dynamic(id, buildFn)`

Report-schema builder. The same API is also available on `ExcelTableSchemaBuilder`.

Generates columns dynamically from the schema-level context object passed at table-build time.

You can also pass `SchemaBuilder.dynamic(id, options, buildFn)`.

| Argument  | Type                                                           | Description                                                                                                                       |
| --------- | -------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `id`      | `string`                                                       | Dynamic scope identifier. Used in `select` and `refs.dynamic("id")`.                                                              |
| `options` | `{ header?: LazyText; condition?: ({ ctx }) => boolean }`      | Optional structural metadata. `condition` omits the whole dynamic scope when it returns `false`.                                  |
| `buildFn` | `(builder: SchemaBuilder<T>, args: { ctx: TContext }) => void` | Called with the current builder and the schema-level context object. The builder can reference earlier outer columns in formulas. |

Selection stays strongly typed at the group and dynamic-scope level. Use the declared id in `select.include` or `select.exclude` to include or exclude the whole scope:

```ts twoslash
import { createExcelSchema, createWorkbook } from "@chronicstone/typed-xlsx";

type Row = { name: string; orgs: number[] };

const schema = createExcelSchema<Row, { orgIds: number[] }>()
  .column("name", { accessor: "name" })
  .dynamic("memberships", (builder, { ctx }) => {
    for (const id of ctx.orgIds) {
      builder.column(`org-${id}`, {
        accessor: (row) => row.orgs.includes(id),
      });
    }
  })
  .build();

createWorkbook()
  .sheet("Users")
  .table("users", {
    rows: [],
    schema,
    context: { orgIds: [] },
    select: { exclude: ["memberships"] },
  });
```

When a schema declares a `Context` generic, `context` is part of the table contract regardless of `select`.

### Shared column-definition behavior

`conditionalStyle` uses the same formula condition DSL as formula columns (`refs.column(...)`, `fx.and(...)`, `fx.or(...)`, comparisons), but it emits native Excel conditional formatting instead of a cell value. It layers on top of the base `style` field.

`conditionalStyle` is visual only. It cannot set worksheet protection flags like `protection.locked` or `protection.hidden`.

`validation` emits native Excel worksheet validation rules. The callback form is the recommended API because it keeps the rule typed and gives access to helpers like `list(...)`, `integer()`, `date()`, `custom(...)`, `between(...)`, `prompt(...)`, and `error(...)`.

`hyperlink` keeps the rendered cell value independent from the hyperlink target. Use `null` or `undefined` in the callback form to omit the hyperlink for a specific cell.

### `SchemaBuilder.build()`

Returns a frozen `SchemaDefinition<T>` object. Safe to share across multiple workbooks.

### `SchemaBuilder.theme(theme)`

Attaches a schema-level `SpreadsheetTheme` to the built schema. Table instances can still override it with `.table(..., { theme })` or explicit `.table(..., { defaults })`.

### `ExcelTableSchemaBuilder.column(id, definition)`

Native Excel table schema builder.

Differences from report schemas:

- no `summary`
- accessors and transforms must resolve to scalar cell values
- intended for flat physical rows only

`conditionalStyle` is also supported on Excel-table schemas. The condition still uses the formula DSL; the schema mode only changes how references compile under the hood.

`validation` is also supported on Excel-table schemas.

### `ExcelTableSchemaBuilder.group(id, buildFn)`

Native Excel table schemas support structural groups too. The callback receives an `ExcelTableSchemaBuilder<T>` so generated child columns stay constrained to Excel-table-safe accessors, transforms, formulas, and totals-row metadata.

### `ExcelTableSchemaBuilder.dynamic(id, buildFn)`

Native Excel table schemas also support runtime-generated flat columns via `dynamic(...)`. The callback receives the schema-level context object as `{ ctx }`.

Formula columns use the same DSL in both schema modes. The schema kind changes how refs compile:

- report schemas compile `refs.column(...)` to regular A1-style cell references
- excel-table schemas compile same-row `refs.column(...)` to structured references like `[@Qty]`

The shipped formula DSL today is:

- `refs.column("col")` for one predecessor cell in the current physical row
- `row.series("col")` for the current logical row's expanded series
- `refs.group("groupId")` for structural scope aggregates
- `refs.dynamic("dynamicId")` for dynamic-scope aggregates

Formula scope is lexical and predecessor-only:

- `refs.column("col")` targets one previously declared column in scope
- inside `group(...)` or `dynamic(...)`, formulas can reference previous outer columns and previous local scope columns
- later formulas can target a previously declared structural group with `refs.group("groupId")`
- later formulas can target a previously declared dynamic scope with `refs.dynamic("dynamicId")`

Available scope aggregations:

- `fx.sum(refs.group("groupId"))`
- `fx.average(refs.group("groupId"))`
- `fx.sum(refs.dynamic("dynamicId"))`
- `fx.count(refs.dynamic("dynamicId"))`

Example:

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";

type Row = { amount: number };

const schema = createExcelSchema<Row>({ mode: "excel-table" })
  .column("amount", { accessor: "amount" })
  .group("derived", (builder) => {
    const withDoubleAmount = builder.column("doubleAmount", {
      formula: ({ row, refs }) => refs.column("amount").mul(2),
    });

    withDoubleAmount.column("tripleAmount", {
      formula: ({ row, refs }) => refs.column("doubleAmount").add(refs.column("amount")),
    });
  })
  .column("derivedTotal", {
    formula: ({ refs, fx }) => fx.sum(refs.group("derived")),
  })
  .build();
```

Excel-table columns also support native totals-row metadata:

- `totalsRow: { label: string }`
- `totalsRow: { label: () => string }`
- `totalsRow: { function: "sum" | "average" | "count" | "countNums" | "min" | "max" | "stdDev" | "var" }`

## `createWorkbook(options?)`

Returns a `Workbook` builder for synchronous, in-memory XLSX generation.

Root options:

- `protection?: true | { password?: string; structure?: boolean; windows?: boolean }`

### `Workbook.sheet(name, options?)`

| Option           | Type                                  | Description                                                            |
| ---------------- | ------------------------------------- | ---------------------------------------------------------------------- |
| `tablesPerRow`   | `number`                              | Number of tables placed side-by-side before wrapping. Default: `1`.    |
| `tableColumnGap` | `number`                              | Blank columns between tables in the same row. Default: `1`.            |
| `tableRowGap`    | `number`                              | Blank rows between table rows. Default: `1`.                           |
| `freezePane`     | `{ rows?: number; columns?: number }` | Freeze the first N rows and/or columns.                                |
| `rightToLeft`    | `boolean`                             | Enable right-to-left sheet direction.                                  |
| `protection`     | `true` or protection options          | Enable worksheet protection and configure protected-sheet permissions. |

Returns a `WorkbookSheet` with a `.table(id, input)` method.

### `WorkbookSheet.table(id, input)`

`id` is a required stable table identifier.

| Field     | Type                                                                                                                                                                                                             | Description                                                                                                             |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| `schema`  | `SchemaDefinition<T>`                                                                                                                                                                                            | Built schema from `createExcelSchema<T>().build()`. Report and Excel-table schemas infer different options.             |
| `rows`    | `T[]`                                                                                                                                                                                                            | Data rows.                                                                                                              |
| `title`   | `string`                                                                                                                                                                                                         | Optional title row rendered above the header.                                                                           |
| `theme`   | `SpreadsheetTheme`                                                                                                                                                                                               | Optional table-level theme. Overrides any schema-level theme, but still sits below explicit `defaults` and cell styles. |
| `select`  | `{ include?: readonly (SchemaColumnId<TSchema> \| SchemaGroupId<TSchema> \| SchemaDynamicId<TSchema>)[]; exclude?: readonly (SchemaColumnId<TSchema> \| SchemaGroupId<TSchema> \| SchemaDynamicId<TSchema>)[] }` | Column, group, or dynamic-scope selection by id.                                                                        |
| `context` | `SchemaContextOf<TSchema>`                                                                                                                                                                                       | Required for any contextful schema built with `createExcelSchema<Row, Context>()`.                                      |

For Excel-table schemas, `.table(id, input)` also accepts root native-table options:

- `name?: string`
- `style?: ExcelTableStyle`
- `autoFilter?: boolean`
- `totalsRow?: boolean`

Returns the `WorkbookSheet` for chaining.

### `defineSpreadsheetTheme(input?)`

Creates a reusable spreadsheet theme with semantic slots.

```ts twoslash
import { defineSpreadsheetTheme } from "@chronicstone/typed-xlsx";

const theme = defineSpreadsheetTheme({
  slots: {
    header: { fill: { color: { rgb: "0B1220" } }, font: { color: { rgb: "F8FAFC" }, bold: true } },
  },
});

const variant = theme.extend({
  slots: {
    title: { fill: { color: { rgb: "450A0A" } } },
  },
});

theme.slot("header");
variant.slot("header", { alignment: { horizontal: "left" } });
```

Built-in themes are also exported as `spreadsheetThemes.classic`, `spreadsheetThemes.slate`, `spreadsheetThemes.forest`, `spreadsheetThemes.rose`, and `spreadsheetThemes.ocean`.

### Output methods

| Method              | Return type     | Description                             |
| ------------------- | --------------- | --------------------------------------- |
| `toUint8Array()`    | `Uint8Array`    | XLSX bytes.                             |
| `toBuffer()`        | `Buffer`        | Node.js Buffer wrapping the XLSX bytes. |
| `writeToFile(path)` | `Promise<void>` | Write the XLSX file to disk.            |

## `createWorkbookStream(options?)`

Returns a `WorkbookStream` builder for incremental, commit-based XLSX generation.

| Option          | Type                                             | Default      | Description                                         |
| --------------- | ------------------------------------------------ | ------------ | --------------------------------------------------- |
| `protection`    | `true` or workbook protection options            | `undefined`  | Workbook-level structure protection metadata.       |
| `tempStorage`   | `"file"` or `"memory"`                           | `"file"`     | Where serialized sheet fragments are spooled.       |
| `tempDirectory` | `string`                                         | OS temp dir  | Directory for file-backed spool.                    |
| `strings`       | `"auto"` ┬Ę `"inline"` ┬Ę `"shared"`               | `"auto"`     | String storage strategy. Overrides `memoryProfile`. |
| `memoryProfile` | `"balanced"` ┬Ę `"low-memory"` ┬Ę `"compact-file"` | `"balanced"` | Convenience preset for string strategy.             |

### `WorkbookStream.sheet(name, options?)`

Accepts the same sheet options as the buffered builder, including layout options and `protection`.

Returns a `WorkbookSheetStream`.

### `WorkbookSheetStream.table(id, options)`

Async. Registers a table and returns a `WorkbookTableStream<T>`.

`id` is a required stable table identifier.

| Field     | Type                                                                                                                                                                                                             | Description                                                                        |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| `schema`  | `SchemaDefinition<T>`                                                                                                                                                                                            | Built schema.                                                                      |
| `select`  | `{ include?: readonly (SchemaColumnId<TSchema> \| SchemaGroupId<TSchema> \| SchemaDynamicId<TSchema>)[]; exclude?: readonly (SchemaColumnId<TSchema> \| SchemaGroupId<TSchema> \| SchemaDynamicId<TSchema>)[] }` | Column, group, or dynamic-scope selection.                                         |
| `context` | `SchemaContextOf<TSchema>`                                                                                                                                                                                       | Required for any contextful schema built with `createExcelSchema<Row, Context>()`. |

For Excel-table schemas, stream table registration also accepts root native-table options:

- `name?: string`
- `style?: ExcelTableStyle`
- `autoFilter?: boolean`
- `totalsRow?: boolean`

### `WorkbookTableStream.commit(batch)`

Async. Serializes the batch and appends it to the sheet spool.

| Field  | Type  | Description                 |
| ------ | ----- | --------------------------- |
| `rows` | `T[]` | Batch of rows to serialize. |

### Output methods

| Method               | Description                                         |
| -------------------- | --------------------------------------------------- |
| `writeToFile(path)`  | Write directly to a file path.                      |
| `pipeToNode(stream)` | Pipe to any `NodeJS.WritableStream`.                |
| `pipeTo(stream)`     | Pipe to a Web Streams `WritableStream<Uint8Array>`. |
| `toNodeReadable()`   | Returns a `NodeJS.ReadableStream`.                  |
| `toReadableStream()` | Returns a Web `ReadableStream<Uint8Array>`.         |

Only one output method may be called per workbook instance.

## `SummaryInput<T>`

The `summary` field accepts either:

- a single `SummaryDefinition<T, TAcc>`
- an array of `SummaryDefinition<T, TAcc>`
- a callback of shape `(summary) => SummaryDefinition[]`

The callback form is the recommended API because it gives you access to helper methods like `summary.cell(...)`, `summary.label(...)`, and `summary.empty()`.

## `SummaryDefinition<T, TAcc>`

| Field      | Type                                            | Description                                                                  |
| ---------- | ----------------------------------------------- | ---------------------------------------------------------------------------- |
| `label`    | `string`                                        | Optional label cell rendered in non-summary columns on the same summary row. |
| `init`     | `() => TAcc`                                    | Returns the initial accumulator value. Called once.                          |
| `step`     | `(acc: TAcc, row: T, rowIndex: number) => TAcc` | Updates the accumulator for each row.                                        |
| `finalize` | `(acc: TAcc) => SummaryCellValue`               | Converts the final accumulator to a cell value.                              |
| `format`   | `string` or `(value) => string`                 | Number format applied to the summary cell.                                   |
| `style`    | `CellStyle` or `(value) => CellStyle`           | Style applied to the summary cell.                                           |
