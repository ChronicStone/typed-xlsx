## `CellStyle`

Applied to data cells via `style`, to header cells via `headerStyle`, and to hyperlink-local style overrides.

```ts twoslash
import type { CellStyle } from "@chronicstone/typed-xlsx";
```

| Field        | Type                  | Description                                      |
| ------------ | --------------------- | ------------------------------------------------ |
| `font`       | `FontStyle`           | Font properties.                                 |
| `fill`       | `FillStyle`           | Background fill.                                 |
| `border`     | `BorderStyle`         | Cell border lines.                               |
| `alignment`  | `AlignmentStyle`      | Text alignment and wrap.                         |
| `numFmt`     | `string`              | Excel number format string.                      |
| `protection` | `CellProtectionStyle` | Worksheet protection flags (`locked`, `hidden`). |

`style` is resolved by the library during export.

`conditionalStyle` does not use the full `CellStyle` surface: worksheet protection flags are intentionally excluded because native Excel conditional formatting cannot toggle `locked` or `hidden`.

## `SpreadsheetTheme`

Created with `defineSpreadsheetTheme(...)`.

```ts twoslash
import type {
  SpreadsheetTheme,
  SpreadsheetThemeSlotName,
  SpreadsheetThemeTokens,
} from "@chronicstone/typed-xlsx";
```

| Field / Method           | Type                                          | Description                                                                                |
| ------------------------ | --------------------------------------------- | ------------------------------------------------------------------------------------------ |
| `tokens`                 | `SpreadsheetThemeTokens`                      | Resolved token set for the theme.                                                          |
| `slots`                  | `Record<SpreadsheetThemeSlotName, CellStyle>` | Resolved slot styles.                                                                      |
| `extend(input?)`         | `(input?) => SpreadsheetTheme`                | Returns a new theme with deep-merged token and slot overrides.                             |
| `slot(name, overrides?)` | `(name, overrides?) => CellStyle`             | Returns one resolved slot as a plain `CellStyle`, optionally merged with ad hoc overrides. |

Default slot names:

- `title`
- `groupHeader`
- `groupHeaderFiller`
- `header`
- `summary`
- `cellBase`
- `cellUnlocked`
- `cellLocked`
- `cellHidden`
- `hyperlink`

Theme precedence during rendering is:

- cell-level style
- explicit table `defaults`
- table-level `theme`
- schema-level `theme`
- built-in library defaults

Built-in themes are exported as `spreadsheetThemes.classic`, `spreadsheetThemes.slate`, `spreadsheetThemes.forest`, `spreadsheetThemes.rose`, and `spreadsheetThemes.ocean`.

## `StructureConditionDefinition<TContext>`

Used by schema node `condition` fields on columns, groups, and dynamic scopes.

```ts twoslash
import type { StructureConditionDefinition } from "@chronicstone/typed-xlsx";
```

```ts
type StructureConditionDefinition<TContext> = ({ ctx }: { ctx: TContext }) => boolean;
```

This is a structural predicate, not a row-level formula condition.

- it only receives `{ ctx }`
- it controls whether the schema node exists in the rendered table shape
- when it returns `false`, the column or scope is omitted entirely

Use `condition` when visibility depends on schema context, not on row data.

## `ConditionalStyleInput<TColumnId, TGroupId>`

Accepted by the `conditionalStyle` field on column definitions.

```ts twoslash
import type { ConditionalStyleInput } from "@chronicstone/typed-xlsx";
```

It can be:

- a callback builder `(conditional) => conditional.when(...)`
- an array of normalized conditional style rules

The callback form is the recommended public API.

### `ConditionalStyleBuilder<TColumnId, TGroupId>`

```ts twoslash
import type { ConditionalStyleBuilder } from "@chronicstone/typed-xlsx";
```

| Method                   | Description                                                                                                                   |
| ------------------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| `when(condition, style)` | Adds one conditional-formatting rule using the formula DSL for `condition` and a visual style overlay for the matching cells. |
| `done()`                 | Returns the normalized rule array. Mostly useful internally or for advanced composition.                                      |

Example:

```ts twoslash
import { createExcelSchema } from "@chronicstone/typed-xlsx";

const schema = createExcelSchema<{ amount: number; quota: number }>()
  .column("amount", { accessor: "amount" })
  .column("quota", { accessor: "quota" })
  .column("attainment", {
    formula: ({ row, refs, fx }) => fx.safeDiv(refs.column("amount"), refs.column("quota")),
    conditionalStyle: (conditional) =>
      conditional.when(({ row, refs }) => refs.column("attainment").lt(1), {
        fill: { color: { rgb: "FEE2E2" } },
        font: { color: { rgb: "991B1B" }, bold: true },
      }),
  })
  .build();
```

## `ValidationInput<TColumnId, TGroupId>`

Passed to the `validation` field on column definitions.

```ts twoslash
import type { ValidationBuilder, ValidationInput, ValidationRule } from "@chronicstone/typed-xlsx";
```

It can be:

- a callback builder `(validation) => validation.integer().between(...)`
- a normalized `ValidationRule<TColumnId, TGroupId>` object

The callback form is the recommended public API.

### `ValidationBuilder<TColumnId, TGroupId>`

| Method                        | Description                                                                |
| ----------------------------- | -------------------------------------------------------------------------- |
| `list(values)`                | Restrict values to a dropdown list.                                        |
| `integer()`                   | Restrict values to integers. Compiles to Excel's `whole` validation kind.  |
| `decimal()`                   | Restrict values to decimal numbers.                                        |
| `date()`                      | Restrict values to dates.                                                  |
| `textLength()`                | Restrict values by string length.                                          |
| `custom(condition)`           | Use the formula DSL for a custom validation condition.                     |
| `between(min, max)`           | Set an inclusive range operator.                                           |
| `notBetween(min, max)`        | Set an exclusive range operator.                                           |
| `eq(value)`                   | Set the `equal` operator.                                                  |
| `neq(value)`                  | Set the `notEqual` operator.                                               |
| `gt(value)`                   | Set the `greaterThan` operator.                                            |
| `gte(value)`                  | Set the `greaterThanOrEqual` operator.                                     |
| `lt(value)`                   | Set the `lessThan` operator.                                               |
| `lte(value)`                  | Set the `lessThanOrEqual` operator.                                        |
| `allowBlank(value?)`          | Allow blanks in the validated range.                                       |
| `showDropDown(value?)`        | Control Excel's dropdown arrow behavior for list validations.              |
| `prompt({ title?, message })` | Configure the input message shown when the cell is selected.               |
| `error({ title?, message })`  | Configure the blocking error alert shown for invalid edits.                |
| `done()`                      | Return the normalized rule object. Mostly useful for advanced composition. |

### `ValidationRule<TColumnId, TGroupId>`

Normalized validation shape used internally and exposed for advanced composition.

| Field          | Type                                                                   | Description                                         |
| -------------- | ---------------------------------------------------------------------- | --------------------------------------------------- |
| `type`         | `"list" \| "whole" \| "decimal" \| "date" \| "textLength" \| "custom"` | Excel validation kind.                              |
| `operator`     | comparison operator                                                    | Optional range/comparison operator.                 |
| `formula1`     | scalar value or formula expr                                           | First comparison input.                             |
| `formula2`     | scalar value or formula expr                                           | Second comparison input for two-value operators.    |
| `source`       | `Array<string \| number>`                                              | Inline source for `list(...)`.                      |
| `allowBlank`   | `boolean`                                                              | Whether blanks should pass validation.              |
| `showDropDown` | `boolean`                                                              | Excel dropdown-arrow behavior for list validations. |
| `prompt`       | `string \| ValidationMessage`                                          | Input message metadata.                             |
| `error`        | `string \| ValidationMessage`                                          | Error alert metadata.                               |

### `ValidationMessage`


| Field     | Type       | Description                     |
| --------- | ---------- | ------------------------------- |
| `title`   | `LazyText` | Optional prompt or error title. |

## `FormulaRowContext<TColumnId, TScopeId>`

The `row` parameter in formula callbacks.

Current shipped members:

- `series(columnId)` for the expanded logical-row series of one column
- `if(condition, whenTrue, whenFalse)` as a convenience alias for `fx.if(...)`

`row` is reserved for row-aware helpers. In the shipped API, predecessor cell references and scope selectors live on `refs`, not on `row`.

## `FormulaRefs<TColumnId, TGroupId, TDynamicId>`

The `refs` parameter in formula callbacks.

Current shipped members:

- `column(columnId)`
- `group(groupId)`
- `dynamic(dynamicId)`

`refs.group(...)` and `refs.dynamic(...)` are the aggregate-scope selectors for later formulas. `refs.column(...)` is the ordinary predecessor-cell selector for current-row formulas.

In the shipped API, `refs.column(...)` remains the most common way to reference predecessor cells inside ordinary row formulas.

## `SchemaDefinition<T, TColumnId = string, TGroupId = never, TDynamicId = never, TSchemaContext = unknown>`

The return type of `createExcelSchema<T>().build()`. Opaque — do not construct manually.

Schemas are frozen after `.build()` and safe to share across multiple workbooks and threads.

`TColumnId` carries the compile-time union of known column ids. `TGroupId` carries the compile-time union of structural group ids. `TDynamicId` carries the compile-time union of runtime dynamic scope ids. These unions are used to type `select.include` and `select.exclude` on the buffered and stream builders.

`TSchemaContext` carries the schema-level `context` object shape declared through `createExcelSchema<Row, Context>()`. If a schema defines `createExcelSchema<Row, { orgs: Org[] }>()`, table inputs for that schema must provide `context.orgs` regardless of selection.

## `CellValue`

The set of primitive values that can appear in a cell.

```ts
type CellValue = string | number | boolean | Date | null | undefined;
```

Array values (`string[]`, `number[]`) are accepted by `accessor` and `transform` to trigger sub-row expansion. The array element type must be `CellValue`.

## `WorkbookStreamOptions`

Options for `createWorkbookStream()`.

| Field           | Type                                             | Default      | Description                                                   |
| --------------- | ------------------------------------------------ | ------------ | ------------------------------------------------------------- |
| `tempStorage`   | `"file"` · `"memory"`                            | `"file"`     | Where serialized sheet fragments are spooled between commits. |
| `tempDirectory` | `string`                                         | OS temp dir  | Directory for file-backed spools.                             |
| `strings`       | `"auto"` · `"inline"` · `"shared"`               | `"auto"`     | String storage strategy. Overrides `memoryProfile`.           |
| `memoryProfile` | `"balanced"` · `"low-memory"` · `"compact-file"` | `"balanced"` | Convenience preset for string strategy.                       |

String strategy meanings:

| Value      | Description                                                                                                                 |
| ---------- | --------------------------------------------------------------------------------------------------------------------------- |
| `"inline"` | Each string written directly into the cell XML. No shared string table in heap. Produces larger files.                      |
| `"shared"` | All strings deduplicated into a shared string table. Smaller output files for repeated values, but the table grows in heap. |
| `"auto"`   | Uses `"balanced"` preset: shared strings enabled, balanced memory/size trade-off.                                           |

`memoryProfile` presets:

| Preset           | Equivalent strings setting | Use when                                     |
| ---------------- | -------------------------- | -------------------------------------------- |
| `"balanced"`     | `"shared"`                 | Default. General purpose.                    |
| `"low-memory"`   | `"inline"`                 | Minimizing heap is the priority.             |
| `"compact-file"` | `"shared"`                 | Minimizing output file size is the priority. |
