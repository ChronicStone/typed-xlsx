# Impeccable Design Context

## Design Context

### Users

TypeScript developers building XLSX export features in their applications. They arrive at the landing page from npm, GitHub, or search — typically evaluating the library in under 60 seconds. They care about type safety, DX, and production readiness. Their context is practical: "Can this replace my hand-rolled Excel code?"

### Brand Personality

**Bold, Technical, Playful** — A developer tool that takes itself seriously on capability but not on presentation. It should feel like the kind of tool a senior engineer built with craft and taste, not a corporate product page.

### Emotional Goals

**Delight & Surprise** — "Wow, this is way nicer than I expected for an XLSX library." The landing page should exceed expectations for what a utility library docs site looks like.

### Aesthetic Direction

- **Visual tone**: Dark-mode is the hero aesthetic (already spot-on). Light mode needs more contrast — cards and sections currently feel white-on-white and lack distinction.
- **Color**: Emerald primary on Zinc neutral. Use emerald at varied opacities for depth. Accent colors (cyan, amber, rose, violet) appear in artifact previews.
- **Typography**: System font stack. Mono for code, eyebrows, and stats. Strong hierarchy from 10px labels to 5.5rem hero titles.
- **Code theme**: Vitesse (dark/light) via Shiki with Twoslash type annotations.
- **Theme**: Equally balanced dark/light, but dark mode sets the standard.

### Design Principles

1. **Contrast creates clarity** — Every section, card, and element must be visually distinguishable from its background in both color modes. No white-on-white, no invisible boundaries.
2. **Motion with purpose** — Scroll animations reveal content progressively. Micro-interactions reward exploration. Nothing animates just because it can.
3. **Typography carries rhythm** — Consistent vertical rhythm, deliberate size steps, and proper breathing room between text blocks. Eyebrow → heading → body should feel like a musical phrase.
4. **Developer credibility through craft** — Code blocks, type annotations, and technical details are first-class citizens, never afterthoughts. Polish the technical content as much as the marketing content.
5. **Surprise within structure** — Playful touches (animations, hover effects, unexpected details) exist within a clean, predictable layout grid. Delight, not chaos.

### Anti-Patterns to Avoid

- Generic SaaS landing page aesthetics (gradient blobs, stock photos, "trusted by" logos)
- Overanimation that feels sluggish or distracting
- Light mode as an afterthought — it must feel intentionally designed
- Flat, monotonous sections with no visual breathing room
